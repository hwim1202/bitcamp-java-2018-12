외부 라이브러리 사용 - CDN(Content Delivery Network) 방식

- 외부 라이브러리를 사용할 때 기존 URL을 그대로 사용한다

[장점]
- 다운로드 할 필요가 없기 때문에 관리가 쉽다
- 웹 브라우저가 사이트 별로 파일을 캐시에 담아두기 때문에 사용자가 다른 시스템을 방문했을 때 
  라이브러리를 이미 다운로드 받았다면 다시 다운로드 받지 않기 때문에 페이징 로딩 속도가 빨라진다
  
[단점]
- 라이브러리 서버와 연결되지 않으면 기능을 온전히 사용할 수 없다
- 라이브러리가 외부에 있기 때문에 사용자가 통제할 수 없다
- 라이브러리가 있는 서버 쪽에서 해킹 코드를 삽입하더라도 막을 방법이 없다
